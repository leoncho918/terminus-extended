# Deploying Terminus on Unraid

This guide walks you through deploying Terminus on an Unraid server using Docker Compose via the terminal. This is the simplest and most reliable approach — Unraid supports Docker Compose natively through the terminal.

---

## Prerequisites

- Unraid 6.12 or later
- Docker installed and running (Unraid includes Docker by default)
- Docker Compose v2 (`docker compose` not `docker-compose`) — included with Docker Desktop/Engine 20.10+
- SSH or terminal access to your Unraid server
- Your Unraid server's local IP address (e.g. `192.168.1.100`)

To verify Docker Compose is available, open the Unraid terminal and run:

```bash
docker compose version
```

If you see `Docker Compose version v2.x.x` you're good. If not, install the Compose plugin via the Unraid Community Applications store.

---

## Step 1 — Choose a Storage Location

Terminus needs a persistent directory for its files. A good place is your Unraid `appdata` share:

```bash
mkdir -p /mnt/user/appdata/terminus
cd /mnt/user/appdata/terminus
```

All commands in this guide should be run from this directory.

---

## Step 2 — Clone the Repository

```bash
git clone https://github.com/usetrmnl/terminus .
```

The `.` at the end clones into the current directory rather than creating a subdirectory.

---

## Step 3 — Run the Setup Script

The setup script creates your `.env` file with auto-generated secrets and attempts to detect your server IP:

```bash
bin/setup docker
```

This will:
- Generate a random `APP_SECRET`
- Generate random `DATABASE_PASSWORD` and `KEYVALUE_PASSWORD`
- Attempt to auto-detect and set `API_URI` to your server's IP

---

## Step 4 — Configure the `.env` File

Open the generated `.env` file and verify the values:

```bash
nano .env
```

The file will look like this:

```env
HANAMI_PORT=2300

API_URI=http://<your-server-ip>:2300
APP_SECRET=<auto-generated>

DATABASE_NAME=terminus
DATABASE_PASSWORD=<auto-generated>
DATABASE_PORT=5432
DATABASE_USER=terminus

KEYVALUE_DATABASE=0
KEYVALUE_PASSWORD=<auto-generated>
KEYVALUE_PORT=6379
```

**Critical settings to verify:**

- `API_URI` — This **must** be the IP address or hostname your TRMNL devices will use to reach the server. If the setup script got it wrong, update it now. Example: `http://192.168.1.100:2300`
- `HANAMI_PORT` — The port Terminus will listen on. Default is `2300`. Change this if you need a different port.

> **Important:** Never change `DATABASE_PASSWORD` or `KEYVALUE_PASSWORD` after first launch without also updating the data volumes or the database itself. If they get out of sync you will lose access to your data.

Save and close with `Ctrl+O`, `Enter`, `Ctrl+X`.

---

## Step 5 — Open the Firewall Port

Make sure port `2300` (or whatever you set `HANAMI_PORT` to) is accessible from your local network. On Unraid this is usually open by default for Docker containers.

If you're accessing from a TRMNL device on the same network, no additional firewall changes should be needed.

---

## Step 6 — Start Terminus

Pull the latest images and start all services:

```bash
docker compose up --pull always --detach
```

This starts four containers:
- `terminus-init-certificates` — one-off container that installs any SSL certificates, then exits
- `terminus-web` — the main Puma web server (runs on your chosen port)
- `terminus-worker` — the Sidekiq background job processor
- `terminus-database` — PostgreSQL 18
- `terminus-keyvalue` — Valkey (Redis-compatible cache/session store)

On first startup, the web container will automatically:
1. Compile frontend assets
2. Run database migrations

This takes about 60–90 seconds. You can watch progress with:

```bash
docker compose logs -f web
```

Wait until you see a line like `Listening on http://0.0.0.0:2300` before trying to access the UI.

---

## Step 7 — Access the Web Interface

Open a browser and go to:

```
http://<your-server-ip>:2300
```

You should see the Terminus registration/login page. Create your account to get started.

---

## Step 8 — Make Terminus Start on Boot

To ensure Terminus restarts automatically when Unraid reboots, add a startup script. Unraid runs `/boot/config/go` on every boot.

Open the file:

```bash
nano /boot/config/go
```

Add the following lines at the end:

```bash
# Start Terminus
cd /mnt/user/appdata/terminus && docker compose up --detach
```

Save and close. Now Terminus will start automatically whenever your server boots.

Alternatively, if you prefer managing this through the Unraid UI, you can use the **User Scripts** plugin (available in Community Applications) to create a startup script with the same command.

---

## Upgrading Terminus

To upgrade to the latest version:

```bash
cd /mnt/user/appdata/terminus
bin/upgrade docker
```

Or manually:

```bash
cd /mnt/user/appdata/terminus
git pull
docker compose pull
docker compose up --detach
```

Watch the logs after upgrading for any migration output:

```bash
docker compose logs -f web
```

## Useful Commands

| Task | Command |
|---|---|
| View all logs | `docker compose logs -f` |
| View web logs only | `docker compose logs -f web` |
| Stop all services | `docker compose down` |
| Stop and remove volumes (destroys data) | `docker compose down -v` |
| Restart a single service | `docker compose restart web` |
| Check container status | `docker compose ps` |

## Reverse Proxy (Optional)

If you want to use a domain name or expose Terminus on port 80/443, set up a reverse proxy in front of it. Unraid users commonly use **Nginx Proxy Manager** (available in Community Applications).

In Nginx Proxy Manager:
1. Add a new **Proxy Host**
2. Set the forward hostname to your Unraid server IP and port to `2300`
3. Add your domain and SSL certificate as desired

Then update `API_URI` in your `.env` file to the new domain (e.g. `https://terminus.yourdomain.com`) and restart:

```bash
docker compose down && docker compose up --detach
```

Your TRMNL devices must be able to reach whatever `API_URI` is set to, so make sure the domain resolves correctly on your network.

## Troubleshooting

**Container won't start / database auth errors**

If you see `password authentication failed for user "terminus"` in the logs, your `.env` passwords are out of sync with the existing database volume. Either:
- Restore the original passwords in `.env`, or
- Delete the database volume and start fresh (you will lose data):
  ```bash
  docker compose down -v
  docker compose up --detach
  ```

**Devices can't connect**

Check that `API_URI` in `.env` exactly matches what the device is configured to use — same IP/hostname and same port. After any change to `.env`, restart the stack:

```bash
docker compose down && docker compose up --detach
```

**Port 2300 already in use**

Change `HANAMI_PORT` in `.env` to a free port (e.g. `2301`) and restart. Also update `API_URI` to match.

**Self-signed certificate errors in extensions**

Add a `CERTIFICATE_URLS` variable to `.env` pointing to your certificate URL(s), then restart:

```env
CERTIFICATE_URLS=https://your-server/your-cert.crt
```
